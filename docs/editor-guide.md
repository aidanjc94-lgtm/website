# Fenland Running Club editor guide

This guide is for committee members and trusted volunteers who update the website through Pages CMS. Pages CMS gives you an editing screen for the files in this repository, so you should not need to edit `.tsx` files or understand pull requests for routine content changes.

## Before you publish

- Use British English.
- Keep the tone friendly, encouraging and inclusive.
- Do not publish private member information.
- Do not publish personal phone numbers, personal email addresses, medical details, safeguarding details or emergency contact information unless the club has explicitly approved them for the public website.
- If a detail is not confirmed, write `TODO:` at the start of the field so it is clearly visible.
- Add useful alt text for every meaningful image.

## Create a news post

1. Open Pages CMS for the repository.
2. Choose **News posts**.
3. Select **New entry**.
4. Fill in:
   - **Title** — the public headline.
   - **Slug** — lower-case words separated by hyphens, for example `frostbite-round-one-report`.
   - **Publication date**.
   - **Summary** — one or two sentences for the news listing and homepage.
   - **Category**.
   - **Published** — switch this off while drafting.
   - **Feature on homepage** — switch this on only for important current updates.
   - **Body** — the full article.
5. Save the post.
6. The GitHub Pages deployment will rebuild the static site after the change is saved to the repository.

The club may add around two news posts per week. Keep posts concise and put long race reports behind clear headings.

## Upload an image

1. In Pages CMS, use an image field or the media library.
2. Upload the image to the **Images** media area.
3. Use a short, safe file name if prompted.
4. Add **Image alt text** that describes the image, for example: `Fenland runners at the start of a Frostbite League race`.
5. Do not upload images unless the club has permission to publish them.

## Upload a document

1. Use the **Documents** media area.
2. Upload PDFs or other approved public documents.
3. Link them from the relevant policy entry.
4. Do not upload private committee papers or member data.

## Add an event

1. Choose **Events and calendar**.
2. Select **New entry**.
3. Add the event title, date label, time label, location, category, status and summary.
4. Set **Published** to true when the details are ready.
5. Use `TODO:` in fields that still need confirmation.

## Update Couch to 5K details

1. Choose **Site settings and pages**.
2. Open **Couch to 5K content**.
3. Update the intro and TODO note with approved programme details.
4. Add or update a related FAQ in **FAQs** if needed.
5. Add a news post when registration opens.

## Update Fenland 10 details

1. Choose **Site settings and pages**.
2. Open **Fenland 10 content**.
3. Update race date, venue, entries, licence information and links only when confirmed.
4. Add or update Fenland 10 FAQs in **FAQs**.
5. Add a news post for important race announcements.

## Update committee, coach and run leader details

1. Choose **People**.
2. Open **Committee members**, **Coaches** or **Run leaders**.
3. Edit the person entry.
4. Add the approved public name, role and a short note.
5. Keep **Published** off if the person has not approved publication.
6. Do not add private contact details or safeguarding-sensitive information.

## Update policies

1. Choose **Policies**.
2. Edit the policy title, summary and status.
3. Upload and link a public document only when it has been approved by the club.
4. Keep old documents out of the public site unless the club has agreed they should remain available.
